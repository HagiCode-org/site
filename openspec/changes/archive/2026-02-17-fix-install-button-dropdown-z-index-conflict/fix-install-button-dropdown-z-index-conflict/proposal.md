# Change: 修复首页安装按钮下拉面板被遮挡问题

## Why

首页 Hero 区域的 InstallButton 组件下拉面板被下方内容区域（FeaturesShowcase 等组件）遮挡，导致用户无法正常查看和操作下拉菜单中的版本选择选项。这是一个 CSS 层叠上下文（stacking context）冲突问题，影响了用户体验和功能可用性。

## What Changes

- 为 InstallButton 组件的下拉面板调整 `z-index` 层级值
- 确保下拉面板在所有页面内容之上正确显示
- 验证 `position` 属性配置，确保 `z-index` 生效
- 保持现有样式和主题（亮色/暗色/农历新年）的兼容性

**影响范围**：
- 文件：`apps/website/src/components/home/InstallButton.module.css`
- 页面：`apps/website/src/pages/index.astro`（首页）
- 组件：InstallButton（full 模式）

## UI Design Changes

无新增 UI 元素，仅修复现有下拉菜单的显示问题。

无代码流程变更，仅 CSS 样式调整。

## Impact

### 受影响的规范
- 无新增或修改的功能规范，这是一个纯样式修复

### 受影响的代码
- `apps/website/src/components/home/InstallButton.module.css` - 调整 `.dropdownMenu` 的 `z-index` 值
- 可能需要检查其他组件是否创建了高层叠上下文

### 验证标准
- 在开发环境测试 `npm run dev:website`
- 确认下拉面板不被任何下方内容遮挡
- 响应式设计在不同屏幕尺寸下均正常工作
- 所有主题（亮色/暗色/农历新年）下均正常显示
- 通过生产构建验证 `npm run build:website`

### 风险评估
- **低风险**：仅修改 `z-index` 值，不影响组件功能和布局
- **兼容性**：现有 CSS 已经为 compact 模式设置了 `z-index: 99999`，可复用相同方案
