## MODIFIED Requirements

### Requirement: 颜色对比度符合标准

系统 SHALL 确保所有主题配色方案符合 WCAG 2.1 对比度标准。亮色主题和暗色主题的文字与背景对比度 SHALL ≥ 4.5:1（WCAG AA 级）。重要文本（如标题、链接）SHALL 达到 AAA 级（≥ 7:1）。

Desktop Hero 组件的次要下载按钮在亮色模式下 SHALL 使用深色文字（`#0F172A`）在浅色半透明背景（`rgba(15, 23, 42, 0.08)`）上，确保对比度 ≥ 7:1。次要下载按钮悬停状态下 SHALL 使用品牌深色文字（`#0060CC`）在品牌色半透明背景（`rgba(0, 128, 255, 0.1)`）上，确保对比度 ≥ 4.5:1。

#### Scenario: Desktop Hero 次要按钮亮色模式对比度验证

- **WHEN** 用户在亮色模式下访问 Desktop 页面
- **AND** 用户的操作系统平台为 Windows，显示为主要按钮
- **THEN** macOS 和 Linux 次要按钮应使用深色文字（`#0F172A`）
- **AND** 次要按钮背景应为深色半透明（`rgba(15, 23, 42, 0.08)`）
- **AND** 次要按钮边框应为深色（`rgba(0, 0, 0, 0.15)`）
- **AND** 文字与背景对比度应 ≥ 7:1（WCAG AAA 级）
- **AND** 用户能够清晰识别所有平台下载选项

#### Scenario: Desktop Hero 次要按钮亮色模式悬停状态

- **WHEN** 用户将鼠标悬停在亮色模式下的次要按钮上
- **THEN** 按钮背景应变为品牌色半透明（`rgba(0, 128, 255, 0.1)`）
- **AND** 按钮文字应变为品牌深色（`#0060CC`）
- **AND** 按钮边框应高亮显示（`rgba(0, 128, 255, 0.3)`）
- **AND** 文字与背景对比度应 ≥ 4.5:1（WCAG AA 级）
- **AND** 应提供清晰的交互反馈

#### Scenario: Desktop Hero 次要按钮下拉切换按钮亮色模式

- **WHEN** 用户在亮色模式下查看次要按钮的下拉切换按钮
- **THEN** 下拉箭头图标应使用深色（`#0F172A`）
- **AND** 下拉按钮背景应为浅色半透明（`rgba(15, 23, 42, 0.06)`）
- **AND** 左侧分割线应为深色（`rgba(0, 0, 0, 0.12)`）
- **AND** 悬停时应显示品牌色高亮效果

#### Scenario: Desktop Hero 次要按钮暗色模式样式保持不变

- **WHEN** 用户在暗色模式下访问 Desktop 页面
- **THEN** 次要按钮应继续使用白色背景（`rgba(255, 255, 255, 0.08)`）
- **AND** 次要按钮文字应保持为白色（`#ffffff`）
- **AND** 次要按钮样式应与修改前保持一致
- **AND** 不应受亮色模式样式修改的影响

#### Scenario: Desktop Hero 次要按钮农历新年主题样式保持不变

- **WHEN** 用户在农历新年主题模式下访问 Desktop 页面
- **THEN** 次要按钮应继续使用新年主题样式
- **AND** 次要按钮样式应与修改前保持一致
- **AND** 不应受亮色模式样式修改的影响

#### Scenario: Desktop Hero 主要按钮样式不受影响

- **WHEN** 用户在任意主题模式下查看主要按钮（用户检测到的平台）
- **THEN** 主要按钮应继续使用渐变背景（蓝-青-绿）
- **AND** 主要按钮文字应保持为白色
- **AND** 主要按钮样式应与修改前保持一致
- **AND** 不应受次要按钮样式修改的影响
